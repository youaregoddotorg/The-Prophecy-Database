# Contributing

Thanks for your interest in contributing to this project. We welcome bug reports, feature suggestions, documentation improvements, and code contributions.

## How to contribute

1. **Check existing issues and pull requests** before opening a new one to avoid duplicates.
2. **Open an issue** for bugs, questions, or feature requests when discussion is needed first.
3. **Fork the repository** and create a branch for your change.
4. **Make your changes** with clear, focused commits.
5. **Submit a pull request** with a concise description of what changed and why.

## Before you start

Before contributing, please make sure you have:

- A GitHub account
- The latest version of the repository
- Any required development tools installed
- Basic familiarity with the project setup and workflow

## Reporting bugs

When reporting a bug, please include:

- A clear description of the problem
- Steps to reproduce it
- Expected behavior
- Actual behavior
- Screenshots, logs, or error messages if available
- Your operating system, browser, or environment details if relevant

## Suggesting features

For feature requests, please explain:

- The problem you are trying to solve
- The proposed solution
- Any alternatives you have considered
- Why the feature would be useful to the project

## Pull request guidelines

Please keep pull requests:

- Focused on a single change or related set of changes
- Well described
- Tested, if applicable
- Updated with relevant documentation changes

If your change affects user-facing behavior, please include examples or screenshots when useful.

## Code style

Please follow the existing style and conventions in the repository. If the project includes linting, formatting, or testing tools, run them before submitting your pull request.

## Commit messages

Use clear, concise commit messages that describe the purpose of the change. Example:

- `fix: handle empty input`
- `docs: update installation instructions`
- `feat: add search filter`

## Review process

Maintainers will review pull requests as time allows. We may request changes before merging. Please be patient and responsive during review.

## Community expectations

Please be respectful and constructive in all discussions. Contributions should follow the project’s code of conduct, if one is provided.

## Questions

If you are unsure where your contribution fits, open an issue first so we can discuss it.
